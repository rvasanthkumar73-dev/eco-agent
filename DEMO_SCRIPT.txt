[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's walk through how Eco-Agent works under the hood. 
First, when a query comes in, it enters the START node and immediately hits our security_checkpoint. 
If any security issues are found, the request is redirected to the security_event_handler which gracefully blocks the input.
If safe, it routes to the primary orchestrator node.
The orchestrator leverages two specialized sub-agents via AgentTools: the tracker_agent, which parses activity logs and performs the carbon footprint mathematics, and the intervention_agent, which generates suggestions to reduce emissions.
Both sub-agents connect to a custom stdio Model Context Protocol server that hosts tools like calculate_emissions and get_green_alternatives.
Once the report is compiled, the flow passes to the approval_node, which pauses the workflow for human confirmation before logging the data.
Finally, the final_output node displays the complete carbon footprint report in the UI.

[2:00 — LIVE DEMO]
(switch to playground at localhost:18081)
Let's see it in action. I'll enter a typical daily log in the playground text box: 
"I drove my car for 25 km today and ate beef for dinner."
I'll submit this request. (pause)
Notice that the orchestrator immediately coordinates. The tracker_agent uses the MCP tool to calculate that driving 25 km in a car emits 4.5 kg of CO2, while a beef dinner emits 15.5 kg of CO2, bringing our daily total to 20 kg.
Then, the intervention_agent suggests alternatives: switching to public transit would save up to 55% of transit emissions, and switching to chicken or a vegetarian meal would save over 80% of dinner emissions. It also recommends forest planting offset projects.
But before saving this to the history, the system pauses. It asks: "Would you like to save this daily activity log and recommendations to your history? (yes/no)"
If I type "yes" and submit, the state is updated, and the final report is displayed.

[3:00 — SECURITY & MCP]
(pause)
Security is core to our design. Our security_checkpoint scans all inputs for PII like emails or phone numbers, and checks for prompt injection keywords. Every single scan is logged to a structured, local JSON audit file for auditability. Additionally, our custom MCP server exposes 4 precise tools that query standard carbon coefficient tables locally to keep computations fast and reliable.
